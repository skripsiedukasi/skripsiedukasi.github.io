1. sebutkan tabel cidr​


Jawaban:

Notasi CIDR adalah representasi ringkas dari alamat IP dan diasosiasikan terhadap awalan perutean. Notasi dibentuk dari sebuah alamat IP, karakter garis miring ("/"), dan sebuah angka desimal. Angka tersebut merupakan jumlah 1 bit dalam topeng subalamat. Nilai besar menunjukkan jaringan lebih kecil.

Alamat IP ditulis berdasarkan standar IPv4 atau IPv6. Alamat tersebut dapat menunjukkan sebuah alamat antarmuka atau alamat yang awal dari seluruh jaringan. Agregasi bit-bit tersebut umumnya disebut sebagai pengenal komputer hos.

Contoh:

192.168.100.14/24 merepresentasikan alamat IPv4 192.168.100.14 dan awalan perutean yang terkait 192.168.100.0, atau sama, topeng subjaringan 255.255.255.0, yang memiliki 24 1-bit leading.

blok IPv4 192.168.100.0/22 merepresentasikan 1.024 alamat IPv4 dari 192.168.100.0 hingga 192.168.103.255.

blok IPv4 2001:db8::/48 merepresentasikan blok alamat IPv6 dari 2001:db8:0:0:0:0:0:0 hingga 2001:db8:0:ffff:ffff:ffff:ffff:ffff.

::1/128 merepresentasikan alamat simpul berulang (loopback) IPv6. Panjang awalannya adalah 128 yang merupakan jumlah bit dalam alamat itu.

Untuk IPv4, notasi CIDR merupakan sebuah alternatif untuk sistem lama dalam merepresentasikan jaringan menurut alamat mulanya dan topeng subjaringan, keduanya ditulis dalam notasi titik-desimal. 192.168.100.0/24 setara dengan 192.168.100.0/255.255.255.0.

Jumlah alamat sebuah subjaringan dapat dihitung sebagai 2panjang alamat – panjang awalan, di mana panjang alamat adalah 128 untuk IPv6 dan 32 untuk IPv4. Misalnya, dalam IPv4, panjang awalan /29 menghasilkan: 232 − 29 = 23 = 8 alamat.

11. diketahui sebuah rumah sakit memiliki sub jaringan awal 192.168.5.0/22. tentukan alokasi subnetwork yang masing masing terdiri dari 50 host dengan menggunakan metode CIDR​


Jawaban:

Untuk mengalokasikan subnetwork dari alamat jaringan 192.168.5.0/22 dengan setiap subnetwork yang memiliki 50 host, kita perlu menggunakan metode CIDR (Classless Inter-Domain Routing).

Dalam notasi CIDR, /22 berarti terdapat 22 bit yang telah dialokasikan untuk jaringan dan 10 bit untuk host. Untuk mencapai setidaknya 50 host per subnetwork, kita perlu memiliki 6 bit host (karena 2^6 = 64 lebih dari 50).

Kita dapat mengalokasikan subnetwork /28 yang memiliki 32 alamat host (2^6 = 64, tetapi kita perlu mengambil dua alamat untuk alamat jaringan dan alamat broadcast). Sehingga, kita dapat mengalokasikan subnetwork sebagai berikut:

- Subnetwork 1: 192.168.5.0/28 (32 alamat host, dari 192.168.5.1 hingga 192.168.5.30)

- Subnetwork 2: 192.168.5.32/28 (32 alamat host, dari 192.168.5.33 hingga 192.168.5.62)

- Subnetwork 3: 192.168.5.64/28 (32 alamat host, dari 192.168.5.65 hingga 192.168.5.94)

- Subnetwork 4: 192.168.5.96/28 (32 alamat host, dari 192.168.5.97 hingga 192.168.5.126)

- Subnetwork 5: 192.168.5.128/28 (32 alamat host, dari 192.168.5.129 hingga 192.168.5.158)

- Subnetwork 6: 192.168.5.160/28 (32 alamat host, dari 192.168.5.161 hingga 192.168.5.190)

- Subnetwork 7: 192.168.5.192/28 (32 alamat host, dari 192.168.5.193 hingga 192.168.5.222)

- Subnetwork 8: 192.168.5.224/28 (32 alamat host, dari 192.168.5.225 hingga 192.168.5.254)

Ini memberikan 8 subnetwork dengan masing-masing memiliki 32 alamat host, yang cukup untuk 50 host yang diperlukan dan memberikan sedikit fleksibilitas untuk pertumbuhan.

Mencari Jumlah Host

Perlu diingat bahwa IPv4 memiliki 32 digit bilangan biner (32 bit). prefix 23 (/23) melambangkan jumlah network ID yang dapat digunakan pada sebuah IP address, dalam kasus ini pembagian network id dan host id adalah seperti ini

11111111.11111111.11111110.00000000 (1 = network id; 0 = host id).

Jumlah angka 0 atau host id adalah 9 angka, karena IP address adalah bilangan berbasis biner, maka kita pangkatkan jumlah host id dengan 2.

[tex]2^{9} = 512[/tex]

Cara cepat:

[tex]32 - 23 = 9\\2^{9} = 512[/tex]

Rentang IP dan IP broadcast:Karena host id berada di oktet ke-3, maka ini adalah prefix kelas B, maka kita harus menjumlahkan prefix dengan 1 byte (8). 23 + 8 = 31Kurangi dengan jumlah total bit dalam sebuah IP, yaitu 32. 32 - 31 = 1Pangkatkan dengan bilangan2.  [tex]2^{1} = 2[/tex]Karena ini adalah IP kelas B, maka kita fokuskan pada oktet ke-3, jika hasil perpangkatan kurang dari pada nilai di oktet ke-3, maka hasil perpangkatan tersebut harus dilipatkan sehingga mendekati atau sama dengan nilai di oktet ke-3, tetapi tidak boleh melebihiLipatkan hasil perpangkatan. 2+2+2+2+2 = 10Untuk IP Network ambil bilangan yang terkecil sehingga kita hanya perlu menambahkan angka 0 di oktet terakhir. IP Network = 10.10.10.0Lipatkan hasil perpangkatan sekali lagi dan dikurangi 1 sehingga kita bisa mendapatkan angka untuk IP broadcast. 10+2 = 12IP Broadcast diambil dari hasil kelipatan tambahan dikurangi dengan 1 dan oktet terakhir diisi dengan 255 . 12 - 1 = 11 sehingga IP broadcast = 10.10.11.255Karena IP Network berada di urutan paling awal daripada IP address dan IP Broadcast berada di urutan terakhir dalam IP address, maka rentang IP valid yang bisa digunakan adalah diantara  IP Network dan IP Broadcast, caranya adalah IP Network + 1 sampai IP Broadcast -1.10.10.10.0 + 1 = 10.10.10.110.10.11.255 - 1 = 10.10.11.254Rentang IP adalah 10.10.10.1 - 10.10.11.254

28. Sebuah perusahaan mendapatkan IP 203.23.23.0 dari ISP. Perusahaan tersebut memiliki 10 departemen. CIDR berapakah yang digunakan?


Jawaban:

IP 203.23.23.0 adalah alamat IP publik dalam jaringan internet dan memiliki subnet mask 255.255.255.0. Jika perusahaan memiliki 10 departemen, maka CIDR yang digunakan adalah /24, yang berarti terdapat 24 bit untuk subnet mask dan sisanya untuk host address

Penjelasan:

Berikut adalah jalannya bagaimana perusahaan membagi alamat IP 203.23.23.0 menjadi subnet untuk 10 departemennya:

Tentukan jumlah host yang dibutuhkan untuk setiap departemen. Misalnya, setiap departemen membutuhkan 50 host.

Tentukan jumlah bit yang dibutuhkan untuk mengidentifikasi subnet. Ini bisa dihitung dengan menggunakan rumus 2^n = jumlah subnet, di mana n adalah jumlah bit yang digunakan untuk subnetting.

Tentukan jumlah subnet yang dibutuhkan. Dalam hal ini, perusahaan membutuhkan 10 subnet karena memiliki 10 departemen.

Hitung jumlah bit yang dibutuhkan untuk subnetting dengan menggunakan rumus log2 (jumlah subnet).

Tentukan jumlah bit yang digunakan untuk subnet mask. Ini adalah 24 bit - jumlah bit yang digunakan untuk subnetting.

Tentukan subnet mask yang sesuai. Subnet mask 255.255.255.0 memiliki 24 bit, jadi perusahaan akan menambahkan bit yang dibutuhkan untuk subnetting untuk menentukan subnet mask yang sesuai.

Tentukan alamat IP subnet untuk setiap departemen. Ini bisa dilakukan dengan membagi alamat IP 203.23.23.0 menjadi subnet yang sesuai dan menetapkan alamat IP host untuk setiap departemen.

Tentukan CIDR untuk setiap subnet. Ini adalah jumlah bit dalam subnet mask dibagi 8 (karena 8 bit diperlukan untuk membentuk 1 byte). Dalam hal ini, CIDR adalah /24 karena subnet mask memiliki 24 bit.

38. Bagaimana cara menentukan nilai cidr suatu ip address?


Jawaban:

Untuk menentukan nilai CIDR (Classless Inter-Domain Routing) suatu alamat IP, Anda perlu melakukan langkah-langkah berikut:

1. Identifikasi alamat IP yang akan ditentukan CIDR-nya. Misalnya, ambil contoh alamat IP: 192.168.0.1.

2. Ubah alamat IP tersebut menjadi format biner. Dalam contoh ini, konversi biner dari 192.168.0.1 adalah: 11000000.10101000.00000000.00000001.

3. Hitung jumlah bit yang digunakan untuk mengidentifikasi jaringan dalam alamat IP tersebut. Dalam format CIDR, ini ditentukan dengan menentukan berapa banyak bit yang tetap pada alamat IP ketika digunakan sebagai jaringan. Misalnya, jika semua digit pada 192.168.0.1 tetap, maka menggunakan 32 bit (IPv4).

4. Hitung berapa banyak bit yang digunakan untuk mengidentifikasi subnet. Ini merupakan bagian dari bit host yang diambil dari total bit 32 (IPv4) dan dihitung dengan cara mengurangi bit yang digunakan untuk mengidentifikasi jaringan. Misalnya, jika menggunakan 24 bit untuk mengidentifikasi jaringan (pada contoh sebelumnya), maka akan tersisa 8 bit untuk mengidentifikasi subnet.

5. Terjemahkan jumlah bit subnet menjadi nilai CIDR. Nilai CIDR didasarkan pada jumlah bit subnet yang digunakan, dan diwakili dengan menambahkan tanda garis miring ("/") diikuti dengan jumlah bit. Misalnya, jika menggunakan 8 bit untuk mengidentifikasi subnet, maka nilai CIDR-nya adalah /24.

Dengan langkah-langkah di atas, Anda dapat menentukan nilai CIDR untuk alamat IP yang diberikan. Perlu diingat bahwa pada IPv4, jumlah bit jaringan dan subnet dapat bervariasi tergantung pada kebutuhan dan konfigurasi jaringan yang digunakan.
