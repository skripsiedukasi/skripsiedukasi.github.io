31. 5. Bagaimanakah indikasi jika Routing dan Remote access sudah aktif?​


Jawaban:

Untuk mengetahui apakah Routing and Remote Access (RRAS) sudah aktif atau belum, ada beberapa indikator yang dapat digunakan.

Melalui Server Manager: Buka Server Manager, klik Local Server, lalu klik Remote Access. Jika RRAS sudah aktif, maka akan terlihat tulisan "Remote Access is currently enabled"

Melalui Control Panel: Buka Control Panel, klik Administrative Tools, lalu klik Routing and Remote Access. Jika RRAS sudah aktif, maka akan terlihat tulisan "Routing and Remote Access" di jendela yang muncul.

Melalui Command Prompt: Buka Command Prompt, ketik "RRAS" lalu enter. Jika RRAS sudah aktif, maka akan terlihat tulisan "Routing and Remote Access Service"

Melalui Services: Buka Services, cari Routing and Remote Access, jika statusnya Running, itu menandakan RRAS sudah aktif.

Semua indikator di atas menunjukkan bahwa RRAS sudah aktif dan dapat digunakan untuk routing dan remote access.

38. kekurangan dari remote access


Kekurangan dari remote access adalah harus terhubung dengan jaringan terlebih dahulu. Selain itu, komputer yang akan diremote juga harus sudah menyala terlebih dahulu dan ketika sedang diremote, komputer tidak bisa digunakan terlebih dahulu. Remote access atau dikenal dengan remote desktop adalah suatu fitur yang dimiliki oleh suatu sistem operasi yang memungkinkan penggunanya untuk mengakses komputer lain melalui komputernya.

Remote access dapat dilakukan dengan aplikasi bawaan sistem operasi atau menggunakan aplikasi remote access lainnya seperti teamviewer. Beberapa fungsi remote access secara umum adalah sebagai berikut

1. Dapat mengendalikan komputer lain dari jarak jauh

2. Dapat mengetahui aktivitas apa saja yang ada pada komputer yang diremote.

3. Membantu user lain dalam menyelesaikan masalah komputernya tanpa harus datang ke lokasi.

Demikian pembahasan mengenai remote access, remote access dikenal juga dengan remote desktop, dengan menggunakan remote access, seseorang dapat mengakses atau menggunakan komputer lain tanpa harus datang langsung dan mengakses secara langsung komputer tersebut.

39. Apakah pengertian dan fungsi dari RDP(Remote Dekstop Protocol)


Remote Desktop Protocol (sering disingkat menjadi RDP) adalah sebuah protokol jaringan yang digunakan oleh Microsoft Windows Terminal Services dan Remote Desktop. RDP dibuat berdasarkan protokol T.120 yang spesifikasinya diumumkan oleh International Telecommunication Union (ITU), yang juga merupakan protokol yang digunakan di dalam perangkat lunak konferensi jarak jauh Microsoft NetMeeting.Klien-klien yang mendukungnya bervariasi, mulai dari sebagian besar sistem operasi Windows 32-bit (termasuk Windows CE dan PocketPC), hingga sistem operasi lainnya, seperti Linux, FreeBSD, UNIXSolaris, dan Apple Mac OS X. Secara default, server yang membuka protokol ini, akan membuka port TCP 3389.

Dengan Remote desktop kita bisa melihat tampilan layar komputer lain (yang diremote) sama seperti di komputer aslinya, dan kita dapat melakukan  perintah sesuai dengan yang kita inginkan.
Misal komputer A dan komputer B. Kita ingin agar komputer A bisa dilakukan remote desktop oleh user dari komputer B. ( komputer A dikendalikan oleh user di  komputer B).
