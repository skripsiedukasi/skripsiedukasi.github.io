Perbedaan dari FTP dan SSH antara lain adalah:

FTP adalah protokol transfer file, sedangkan SSH adalah protokol jaringan.FTP secara inheren aman, sedangkan SSH secara inheren aman.FTP hanya memungkinkan kontrol file, sedangkan SSH memungkinkan berbagai aplikasi.FTP dapat mentransfer file hanya sementara SSH memungkinkan untuk terhubung ke posisi bahkan jika itu adalah mungkin untuk mentransfer file dengan SCP.SSH menggunakan enkripsi ketika membangun koneksi antara server dan klien sedangkan FTP tidak mengandung indikasi tentang keamanan karena dirancang di usia di mana keamanan tidak benar-benar penting.SSH adalah aman secara default sebagai lawan FTP

2. jelaskan hubungan antara ftp secure dengan command ssh​


FTP Secure (FTPS) adalah protokol transfer file yang menggunakan enkripsi untuk melindungi keamanan transfer data. FTPS menggunakan keamanan SSL/TLS untuk melindungi data saat transit antara client dan server. Ini memastikan bahwa data yang ditransfer tidak dapat dicegat atau dimodifikasi oleh pihak yang tidak berwenang.

Command SSH (Secure Shell) adalah protokol jaringan yang dirancang untuk mengamankan komunikasi antara dua perangkat, seperti komputer client dan server, melalui jaringan yang tidak aman. Melalui SSH, pengguna dapat mengakses dan mengendalikan perangkat dari jarak jauh dengan aman. SSH menggunakan enkripsi simetris dan asimetris untuk mengamankan koneksi dan melindungi data dari serangan pihak ketiga.

Hubungan antara FTP Secure dan command SSH adalah bahwa keduanya dirancang untuk meningkatkan keamanan dalam proses transfer data atau akses ke perangkat yang aman.

Meskipun keduanya berfokus pada keamanan, FTPS dan SSH berbeda dalam cara mereka bekerja. FTPS beroperasi dalam lapisan aplikasi dan memberikan enkripsi end-to-end untuk transfer file melalui protokol FTP. Sementara itu, SSH beroperasi lebih rendah dalam lapisan transportasi dan memberikan enkripsi end-to-end untuk mengamankan koneksi jaringan, serta memberikan akses baris perintah ke perangkat yang aman.

Dalam beberapa kasus, SSH dapat digunakan untuk mengamankan akses ke server FTPS. Dalam skenario ini, pengguna akan terlebih dahulu menggunakan SSH untuk terhubung ke server secara aman, dan kemudian mereka dapat menggunakan protokol FTPS untuk mentransfer file dengan aman melalui koneksi yang dienkripsi oleh SSH.

Dengan menggunakan kedua teknologi ini, pengguna dapat menjaga keamanan data selama transfer file dan akses jarak jauh mereka, memastikan bahwa informasi sensitif tetap terlindungi dari potensi serangan atau pengintaian.

5. Apa itu SCP?, dimana lokasi SCP Foundation?, apa tujuan SCP?​


SCP Foundation atau Secure, Contain, and Protect adalah sebuah organisasi fiktif rahasia yang dipercayakan pemerintah untuk melacak, menangani, mengamankan, dan menampung objek-objek, lokasi, hingga fenomena anomali yang melanggar hukum alam. Lokasi SCP Foundation dirahasiakan dari publik agar tidak menimbulkan kepanikan massal. Fasilitas penampungan objek-objek SCP sendiri juga dikenal sebagai Museum SCP.Tujuan pendirian SCP sendiri adalah untuk memastikan fenomena dan objek anomali yang terdeteksi di Bumi dapat diamankan dan diteliti dalam fasilitas khusus agar tidak membahayakan manusia.

Pembahasan

SCP awalnya merupakan sebuah proyek fiksi kolaboratif berbasis website yang sempat menghebohkan jagat dunia maya. Situs yang dirilis mulai tahun 2008 itu sempat menuai perdebatan tentang keasliannya. Meskipun banyak yang menganggap SCP adalah sebuah organisasi nyata, tapi SCP hanyalah sebuah proyek fiktif yang mengangkat tema creepypasta bergenre fiksi ilmiah dan fantasi urban.

Sebagai sebuah fiksi, SCP merupakan series cerita yang patut dipuji karena pengemasan ceritanya yang unik. Alih-alih menggunakan format penulisan fiksi normal, SCP Tale dikemas dalam format penulisan ilmiah atau laporan hasil observasi. Setiap objek yang diceritakan diberi kode-kode nomor seperti objek penelitian (Contoh: SCP-001). Dalam cerita juga dijabarkan prosedur pengamanan serta efek samping yang ditimbulkan jika manusia berada dalam satu ruangan dengan objek SCP. Beberapa detail juga diberikan sensor informasi rahasia yang menambah kesan keaslian dari seri fiksi SCP ini.

FTP adalah singkatan dari File Transfer Protocol atau Protokol Transfer File. Mari kita uraikan tentang hal ini. FTP merupakan seperangkat aturan yang digunakan perangkat dalam jaringan TCP/IP (internet) untuk memindahkan file. Saat Anda memakai internet, Anda sebenarnya menggunakan berbagai jenis protokol. Untuk menjelajahi internet, Anda memakai HTTP. Untuk mengirim dan menerima pesan instan, Anda menggunakan XMPP. FTP secara sederhana merupakan protokol yang digunakan untuk memindah-mindahkan file.

Fungsi FTP SERVER:

FTP memfasilitasi setiap user untuk melakukan transfer data secara dua arah. Artinya, jika FTP digunakan dalam sebuah perusahaan maka setiap pemimpin perusahaan mampu mengirimkan file kepada karyawan dan sebaliknya hanya dengan menggunakan server yang sama.

Dapat melakukan pertukaran file antarkomputer dengan mudah meskipun file tersebut memiliki ukuran besar.

Adanya FTP membuat pemilik website dapat melakukan backup website dengan mudah.

Progress perpindahan data tidak akan hilang walaupun sambungan terputus.

Transer data atau file dapat dilakukan dengan mudah dan terorganisir.

Dapat melakukan indirect maupun implicit remote computer.

FTP menyediakan transfer data yang reliabel dan efisien.

Pengertian FTP CLIENT:

FTP Client adalah sebuah software yang berfungsi untuk melakukan request ke layanan (service) FTP Server, FTP Client memiliki interface langsung ke client untuk upload dan download file (pertukaran data) pada FTP Server. Tidak hanya download dan upload FTP Client juga bisa melakukan Rename, Delete, Edit Permission dll.

Fungsi FTP CLIENT:

FTP Client akan melindungi pengguna komputer dari berbagai sistem file storage antar host yang saling bertukar informasi. Pengguna dapat memanfaatkan FTP secara indirect maupun implicit remote computer

Dan penulis akan lebih memfokuskan pembahasannya hanya pada server PROFTPD saja.

VSFTPD adalah FTP server default untuk Ubuntu, CentOS, Fedora, dan Red Hat. Hal ini membuatnya sangat mudah untuk menginstal dibandingkan dengan proftpd, tetapi tidak hadir semudah dari file konfigurasi. vsftpd juga dilaporkan lebih aman.

27. Jelaskan bagaimana proses ssh menyampaikan data dari ssh client sampai ssh server?


Proses ssh menyampaikan data dari ssh client sampai ssh server adalah ketika Client dan server setuju untuk menggunakan sesi SSH tertentu, clien meminta public key kemudian client dan server menyetujui alogaritma dan membentuk suatu session key. Kemudian server mendecrypt session dan mengirimkannya kembali ke client untuk verifikasi. Terakhir, pemakai mengotentikasi dirinya ke server didalam aliran data terenskripsi dalan session key dan clien selanjutnya bekerja secara interaktif pada server mentransfer file ke atau dari server.  

Pembahasan  

SSH adalah protokol aman yang digunakan sebagai sarana utama untuk menghubungkan ke server Linux dari jarak jauh. Menyediakan antarmuka berbasis teks dengan pemijahan shell jarak jauh. Setelah tersambung, semua perintah yang di ketik di terminal lokal dikirim ke server jarak jauh dan dijalankan di sana.

Cara paling umum menghubungkan ke server Linux remote adalah melalui SSH. SSH adalah singkatan dari Secure Shell dan menyediakan cara yang aman dan aman untuk mengeksekusi perintah, membuat perubahan, dan mengonfigurasi layanan dari jarak jauh. Ketika terhubung melalui SSH, maka akan masuk menggunakan akun yang ada di server jauh.

Cara kerja SSH

Ketika   terhubung melalui SSH, maka akan jatuh ke sesi shell, yang merupakan antarmuka berbasis teks di mana  dapat berinteraksi dengan server . Selama sesi SSH  , setiap perintah yang di ketikkan ke terminal lokal   dikirim melalui terowongan SSH terenkripsi dan dijalankan di server.

Koneksi SSH diimplementasikan menggunakan model client-server. Ini berarti bahwa untuk koneksi SSH yang akan ditetapkan, mesin remote harus menjalankan perangkat lunak yang disebut daemon SSH. Perangkat lunak ini mendengarkan koneksi pada port jaringan tertentu, mengotentikasi permintaan koneksi, dan memunculkan lingkungan yang sesuai jika pengguna memberikan kredensial yang benar.

Komputer pengguna harus memiliki SSH klien. Ini adalah perangkat lunak yang mengetahui cara berkomunikasi menggunakan protokol SSH dan dapat diberikan informasi tentang host jarak jauh untuk terhubung, nama pengguna yang digunakan, dan kredensial yang harus diloloskan untuk mengotentikasi. Klien juga dapat menentukan detail tertentu tentang jenis koneksi yang ingin mereka buat.

Klien umumnya mengotentikasi baik menggunakan sandi (kurang aman dan tidak disarankan) atau kunci SSH, yang sangat aman.

Login sandi dienkripsi dan mudah dimengerti bagi pengguna baru. Namun, bot otomatis dan pengguna jahat sering kali berulang kali mencoba mengautentikasi ke akun yang memungkinkan proses masuk berbasis kata sandi, yang dapat mengarah pada kompromi keamanan. Untuk alasan ini, kami menyarankan untuk selalu menyiapkan autentikasi berbasis kunci SSH untuk sebagian besar konfigurasi

Kunci SSH adalah satu set kunci kriptografi yang cocok yang dapat digunakan untuk otentikasi. Setiap set berisi kunci publik dan pribadi. Kunci publik dapat dibagikan secara bebas tanpa kepedulian, sementara kunci privat harus dijaga dengan waspada dan tidak pernah diperlihatkan kepada siapa pun.

Untuk mengotentikasi menggunakan kunci SSH, pengguna harus memiliki pasangan kunci SSH di komputer lokal mereka.

Menghasilkan Pasangan Kunci SSH

Menghasilkan pasangan kunci publik dan pribadi SSH baru di komputer lokal   adalah langkah pertama menuju autentikasi dengan server jauh tanpa kata sandi. Kecuali ada alasan kuat untuk tidak melakukannya,   harus selalu mengotentikasi menggunakan kunci SSH.

Sejumlah algoritma kriptografi dapat digunakan untuk menghasilkan kunci SSH, termasuk RSA, DSA, dan ECDSA. Kunci RSA umumnya lebih disukai dan merupakan jenis kunci default.

34. Apa pengertian dan fungsi dari HTTP,FTP,SMTP,IMAP4 dan POP3??


HTTP (Hypertext Transfer Protocol)
Adalah sebuah protokol untuk melakukan akses antara client dan server. Client dan server masing-masing salaing menerima dan menjawab request keduanya.

FTP (File Transfer Protocol)
Adalah suatu protokol yang berfungsi untuk tukar-menukar file dalam suatu network yang menggunakan TCP koneksi bukan UDP. 

SMTP (Simple Mail Transfer Protocol)
Adalah seperangkat pedoman komunikasi yang memungkinkan perangkat lunak untuk mengirimkan email melalui Internet.

IMAP4 (Internet Message Access Protocol 4)
Adalah hampir sama dengan POP3. Bedanya, IMAP4 bisa dikonfigurasi untuk mendownload hanya header saja yang berisi nama atau alamat pengirim beserta subjectnya dan user dapat memilih email mana yang mau di download untuk dibaca yang berfungsi untuk meminimalkan biaya download data.

POP3 (Post Office Protocol version 3)
Adalah sebuah protokol mail standar yang digunakan untuk menerima email dari server jauh ke klien email lokal.
